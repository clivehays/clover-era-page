üéØ Onboarding Overview

This template provides a comprehensive framework for conducting effective team onboarding sessions. The process is designed to be conversational and focused on solving real problems managers and teams face daily.

Session Duration: 30-40 minutes

Keep the tone conversational and avoid making this feel like a product pitch. Frame it as a tool to solve a real problem the manager and team experience daily.

Meeting Flow at a Glance

Part 1: Context & Backstory 5-7 min

Set the stage with the problem, solution, and why this matters

Part 2: How the App Works 8-10 min

Demo the app features and user experience

Part 3: Download & Enrollment 3-5 min

Get everyone set up in real-time

Part 4: Manager Dashboard 6-8 min

Show how insights turn into action

Part 5: Q&A Session 8-10 min

Address questions and concerns

Part 6: Closing 2-3 min

Set expectations and next steps

Key Messages to Reinforce Throughout

  • Completely anonymous (say this at least three times)
  • Takes less than 30 seconds daily
  • Designed for managers who lack tools despite responsibility
  • Focus on small, incremental improvements
  • Real-time insights vs. rearview mirror surveys
  • Action-oriented, not just data collection

üí¨ Part 1: Context and Backstory 5-7 minutes

Opening

  • Brief introduction of yourself and Clover ERA
  • Acknowledge the team and their manager

The Problem

  • 70% of transformations fail (McKinsey data)
  • 79% of people are not engaged at work
  • Clarify what engagement actually means (not just happiness or satisfaction)
  • 60% of workers show up, do their job, collect paycheck, but aren't invested in outcomes
  • Traditional surveys provide rearview mirror data, not real-time insights

The Gap

  • Managers are expected to look after people but given almost no tools or training
  • This app helps managers understand and improve the team environment
  • Focus on incremental action and small daily improvements

üì± Part 2: How the App Works 8-10 minutes

Key Features

  • ONE question per day
  • Takes less than 30 seconds
  • Completely anonymous (emphasize multiple times)

Anonymity Details

  • No names collected
  • No device IDs tracked
  • Only team code for association
  • Questions rotate through CLOVER elements

User Experience Walkthrough

  1. Show the daily question format
  2. Demonstrate emoji-based response scale
  3. Explain dropdown reason selection
  4. Show optional free text field (remind about anonymity)
  5. Preview the micro-action that appears after submission

Live Demo (share phone screen)

  1. Display enrollment screen
  2. Show "Enroll as Team Member" option
  3. Enter team code process
  4. Navigate through terms and conditions
  5. Show main question interface
  6. Demonstrate settings:
    • Notification time customization
    • Light/dark theme toggle
    • Work days selection
  7. Mention upcoming features if applicable (multiple team enrollment)

‚¨áÔ∏è Part 3: App Download and Enrollment 3-5 minutes

Installation Instructions

  1. Direct team to App Store or Google Play Store
  2. Display QR code on screen
  3. Share team enrollment code (drop in chat)
  4. Guide through first-time setup if anyone has questions

Immediate Testing

Ask team members to download and enroll in real-time:

  • Verify first question appears
  • Confirm notifications are set appropriately
  • Address any technical issues

üìä Part 4: Manager Dashboard Overview 6-8 minutes

Dashboard Components

  • Overall engagement score display
  • Response count and timeframe (typically 90 days)
  • Individual scores for each CLOVER element
  • Identification of problem areas (show example like enablement issues)

Reporting Cycle

Bi-weekly AI-Generated Reports

  • Top themes identification
  • Specific recommendations for addressing issues
  • Actionable next steps

Action Hub

  • How actions get created and tracked
  • Worksheet library: 150 worksheets, approximately 30 minutes each
  • Micro-actions and mini-actions: 600+ options
  • Action completion tracking and feedback loop
  • Connection to manager KPIs around employee well-being

Data Anonymity Assurance

Emphasize Privacy

  • Show that only aggregated data appears in reports
  • Demonstrate that individual responses are not visible
  • Explain how themes emerge from collective input

‚ùì Part 5: Q&A Session 8-10 minutes

Common Questions to Address

Purpose and Use of Feedback

Explain how feedback drives actionable insights and improvements

Multiple Team Enrollment

  • Scenarios for users on multiple teams
  • Complexity considerations
  • Data clustering and anonymity verification

Question Consistency

  • Everyone gets the same daily question
  • Rationale: Creates conversation opportunities
  • Team structure and code management
  • Functional vs. project team alignment

Manager Commitment

Set Clear Expectations

  • Emphasize that this requires action, not just data collection
  • Set expectation for manager to implement recommendations
  • Frame as ongoing improvement process
  • Distinguish from passive annual surveys

üé¨ Part 6: Closing 2-3 minutes

Next Steps

  1. First report available after two weeks of data
  2. Manager will share findings and actions with team
  3. Open door for feedback on app experience
  4. Contact information for support

Final Reminders

  • Answer daily for best results
  • Complete anonymity maintained
  • Takes only 30 seconds
  • Small actions compound over time
